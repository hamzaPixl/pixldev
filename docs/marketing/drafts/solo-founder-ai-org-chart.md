_I run a multi-product studio alone. The org chart is real. It's just rows in a database instead of names on a door._

---

People ask how big the team is.

The honest answer is one, plus a payroll of software that doesn't come to standup. That sounds like a joke until you look at how the work is actually divided.

I wrote before that the company is an API call. This is the part underneath: if a company is provisioned like infrastructure, then the org chart is data, and a solo founder can staff one without hiring anyone.

## An employee is a row

The most useful decision I made wasn't a model choice. It was to treat an agent exactly like a hire.

A human employee gets an identity, credentials, a budget, and a paper trail. So does an agent. Same table, different `type`.

→ **Identity**: the agent exists as a row, with a name and a role, not an anonymous script.
→ **Credentials**: it holds its own key, so its actions are its own, not the company's shared login.
→ **A trail**: every call it makes is attributable later, to a query rather than a hunch.

Once an agent has those three, the difference between it and a contractor is mostly that it never asks for a raise. The org chart stops being a slide and becomes something I can query.

## Budgets make it real

An employee you can't budget isn't an employee. It's an outage waiting for a bill.

So each agent gets an allowance, the same way a person gets one. The plan is that every model call routes through one gateway that checks the balance, spends against it, and says no when it's gone. Budget spent, agent stops, the way a person stops when the month's number is used up.

That's what turns "I have agents" into "I have a team." Not autonomy. **Accountability.** A team is a set of people who each own a budget and a trail, and I'm building agents to clear the same bar.

## Why this scales one person

Because the expensive part of a company was never the labor. It was the coordination.

With agents as rows, the coordination is contracts and queries, not meetings. I can ask which agent spent what, on which product, this week, and get an answer instead of a shrug. One person can run several products because the org chart is legible enough to audit from a laptop.

The studio isn't big. It's just wired so that being small doesn't mean being blind.

## The honest state

Here's the part I won't oversell.

The identity rows are real. The trail is partial. **The gateway that enforces the budgets barely exists yet,** which means right now my agents have job titles but no working spending limit, and the ledger cheerfully reports zero. An org chart with everyone marked as free labor. Charming, and untrue.

So this is a vision post with real foundations, not a finished company. The table exists. The enforcement is being built. Humans, meaning me, still keep the last call, because accountability shouldn't be automated even when the work is.

The team is one person and a table. The table is the point.
